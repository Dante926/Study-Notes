// 核心模块之Events
/* 
    ①通过EventEmitter类实现事件统一管理
    ②实际开发中引入该模块的概率也不大，因为Node本身是基于事件驱动来实现的异步操作
    ③事件驱动的底层就是通过EventEmitter类实现事件统一管理，很多核心模块就继承了这个类。这意味着他们本书就具备事件注册和发布的能力
    // 具体实现
    ④Nodejs是基于事件驱动的异步操作架构，内置events模块。
    ⑤events模块提供了EventEmitter类(这个类的实力对象就具备注册、发布、删除等一系列模型机制)。Nodejs中很多内置核心模块继承EventEmitter类

    EventEmitter常见API
    1.on：添加当事件被触发时调用的回调函数(具体作用就是可以注册一个或多个事件监听器，就是事件被执行时的回调操作)
    2.emit：触发事件，按照注册的序同步调用每个事件监听器（如果这个事件之前被订阅过，也存在对应的监听操作。那就按照顺序来执行具体的监听）
    3.once:添加当事件在注册之后首次被触发时调用的回调函数(用once注册的监听器只会在第一次被触发后执行回调.然后该监听就会被删除)
    4.off:移除特定的监听器

*/
