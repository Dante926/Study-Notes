```js
// 模块化历程
/* 
    ①前端开发为什么需要模块化 -> web开发工作量提升，组件化前端开发
    ②传统开发常见问题：命名冲突和污染 代码冗余，无效请求多 文件间的依赖关系复杂
    ③项目难以维护不方便复用
    ④模块就是小而精且利用维护的代码片段
    ⑤利用函数、对象、自执行函数实现分块

    常见模块化规范
    1.CommonJS规范：规定每个JS文件是一个模块，每个模块都有自己的作用域，在这个作用域里存在变量、函数、类 这些都是私有的，外部不可以直接使用。它提供了exports、module.exports来选择性导出变量。若其他模块想要使用这些数据可以使用require来进行加载。（Nodejs就是使用的CommonJS规范）CommonJS是同步加载的，不适合浏览器中使用。
    2.AMD规范：异步加载（）提供了define、require来选择性加载模块。代表有require.js  
    3.CMD规范：整合上诉两个规范的特点，专门用于实现浏览器平台下异步模块的加载，经典代表C.js
    4.ES6模块化规范：
    1.export：导出模块，export default导出默认模块，export{}
    2.import：导入模块，import default from '模块名'

    总结：
    模块化是前端走向工程化中的重要一环
    早期JS语言层面没有模块化规范
    CommonJS、AMD、CMD、ES6都是模块化规范
    ES6中将模块化纳入标准规范
    当下常用规范是CommonJS与ES6(ESM)
*/

```
